# T09 联调、验收与发布收口

## 任务目标

完成端到端联调、回归验证、缺陷收口和上线准入评审，确保新功能可以稳定交付。

## 任务范围

- 环境检查
- 核心链路测试
- 异常链路测试
- 性能观察
- 文档收口
- 发布准入结论

## 不在范围内

- 不再新增功能
- 不做架构级返工

## 测试范围

### 功能主链路

- 短音频单段
- 长音频自动分段
- 尾帧连续性
- 最终拼接
- 本地落盘
- 自动清理

### 异常链路

- ComfyUI 断连
- `ffmpeg` 缺失
- 某段执行失败
- 页面刷新恢复
- 从失败段重试

### 性能观察

- 30 秒音频任务
- 60 秒音频任务
- 多任务排队

## 实施步骤

### Step 1 准备测试素材

- 3 组短音频
- 3 组长音频
- 2 组复杂音频

### Step 2 跑核心用例

- 按 PRD 验收项逐条执行

### Step 3 跑异常用例

- 模拟中断、失败和恢复

### Step 4 缺陷分级

- P0 阻断交付
- P1 影响核心体验
- P2 可带已知问题上线

### Step 5 发布准入

- 输出是否可上线的最终结论

## 交付物

- 测试记录
- 缺陷清单
- 已知限制列表
- 发布建议

## 验收标准

- PRD 中的核心验收项全部通过
- 无 P0 缺陷
- P1 缺陷有明确处理结论
- 已知限制已写入文档
