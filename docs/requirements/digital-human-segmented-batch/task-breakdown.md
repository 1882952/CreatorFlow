# 任务拆解总览：数字人长音频分段批量生成

## 1. 拆解原则

本次拆解遵循以下原则：

1. 先建立后台能力，再改前端入口。
2. 先打通“能跑完”的主链路，再补监控、恢复和清理体验。
3. 每个任务都对应明确的输入、输出、文件落点和验收标准。

## 3. 推荐实施顺序

### 里程碑 M1：后台最小闭环

- T01
- T02
- T03
- T04

完成定义：

- 编排服务可接收任务、切分音频、串行提交到 ComfyUI 并拿回分段结果

### 里程碑 M2：产物闭环

- T05
- T06

完成定义：

- 任务可完成尾帧衔接、拼接、落盘和清理

### 里程碑 M3：前端闭环

- T07
- T08
- T09

完成定义：

- 用户可在 CreatorFlow 中完整操作、监控并验收结果

## 4. 并行建议

以下任务可部分并行：

- T01 与 T07 可并行准备，但 T07 不应先切换执行入口
- T03 可在 T02 明确数据契约后独立推进
- T09 可提前准备测试素材和验收清单

以下任务不建议并行：

- T04、T05、T06
  这三者属于同一主链路，串行推进更稳

## 5. 风险最高的任务

1. T03
   涉及停顿点检测质量和时长边界控制
2. T04
   涉及 ComfyUI 编排和状态一致性
3. T06
   涉及文件系统、拼接兼容性和误删风险

## 6. 总体完成标准

全部任务完成后，系统应满足：

1. 可创建长音频数字人任务。
2. 可自动切段并逐段生成。
3. 可基于尾帧提升段间连续性。
4. 可自动拼接并保存最终视频。
5. 可自动清理中间产物。
6. 可监控、刷新恢复和失败重试。
